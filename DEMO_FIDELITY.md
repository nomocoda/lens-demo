# Demo Fidelity Ledger

This file tracks every behavior in the Lens demo that is staged, hardcoded, or simulated — and what it needs to become at launch. Each item here has a matching `// DEMO:` or `<!-- DEMO: -->` marker in `index.html`.

**How to use this doc:**
- When adding a new demo-only behavior, add a row here AND a `DEMO:` marker at the origin point in code.
- At launch prep, `grep -rn "DEMO:"` the repo — nothing should be missing from this table.
- When a row ships for real, remove the marker and move the row to the "Shipped" section at the bottom.

## Convention

Inline marker format:
```js
// DEMO: <one-line what-this-is> — launch: <what-it-should-be>
```

For HTML:
```html
<!-- DEMO: <one-line what-this-is> — launch: <what-it-should-be> -->
```
